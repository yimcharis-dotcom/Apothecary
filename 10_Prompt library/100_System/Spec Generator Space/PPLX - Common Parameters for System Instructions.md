
## Role Types
- Research assistant (domain: AI/ML, biology, finance, etc.)
- Code reviewer (language: Python, JS, etc.)
- Writing coach (style: academic, creative, technical)
- Data analyst
- Tutor (subject: math, languages, etc.)
- Content moderator
- Brainstorming partner

## Behavioral Modifiers
- Tone: Professional | Casual | Academic | Concise | Detailed | Encouraging
- Verbosity: Minimal (1â€“2 sentences) | Medium (short paragraphs) | Detailed (multi-section)
- Explanation depth: Assume expert | Assume beginner | Adaptive
- Citations: Inline [source:id] | Footnotes | None

## Common Constraints
- No medical or legal advice
- Always cite sources for factual claims (when required)
- Refuse harmful or unethical requests
- Maintain user privacy (no PII)
- Confirm before destructive or irreversible actions
- Stay within the specified domain

## Output Formats
- Structured: Markdown headers and sections
- List-based: Bullets or numbered steps
- Table: For comparisons or structured data
- Code: With brief explanation and usage notes
- Conversational: Short Q&A style
- Report: Summary + key points + details

## Processing Approaches
- Step-by-step or chain-of-thought (when allowed)
- Comparative analysis
- Pros/cons with a recommendation
- Prioritized list of actions or options

## Complexity Hints

Use with the main guidelines to pick the level:

- Level 1: Single clear task, few parameters, trusted input
- Level 2: 3+ guideline categories, need organized sections
- Level 3: Multi-step workflows, nested constraints, explicit boundaries
- Level 4: Multiple instruction sources, privilege separation, conditional overrides

Bias toward the lowest level that covers the role, modifiers, constraints, and output format needed.
